# Interview App Rubric Framework

## Evaluation Direction

Interview App should evaluate practical technical judgment, not only correctness.
The rubric system should explain why an answer is strong or weak, and it should stay reusable across multiple domains later.

## First MVP Rubric Target

The first MVP rubric is for the individual practice mode flow:

- one question
- one answer
- one evaluation result

The first version can stay simple, but it must still be interpretable.

## Core Rubric Dimensions

- response completeness
- reasoning quality
- clarity of communication
- prioritization / tradeoff judgment
- practical awareness of risk

## Rubric Structure

The rubric should support:

- required points
- optional strong signals
- common weak patterns
- explanation text for each criterion

## First MVP Evaluation Output

The first evaluation output should include:

- overall result summary
- strengths
- gaps
- next improvement advice

## Important Rubric Principle

Do not treat one exact answer as the only correct answer.
The rubric should compare responses to good answer patterns and missing coverage, not exact wording.

## Future Expansion

The same rubric framework should be reusable for:

- code review
- architecture / system design
- debugging
- Excel scenarios
- Splunk scenarios
- Power BI scenarios
